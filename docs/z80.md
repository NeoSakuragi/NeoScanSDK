# KOF96 Z80 Sound Driver — Complete Analysis

## Driver Identity
- **Version**: Sound Driver Ver 0.1, dated 96/07/03
- **Signature**: `Sound Driver Ver 0.1 96/07/03 To SNK`
- **M-ROM**: 128KB (131072 bytes), Z80 at 4MHz, 2KB RAM at $F800-$FFFF

## Architecture

### Memory Map
| Z80 Address | Size | Contents |
|------------|------|----------|
| $0000-$7FFF | 32KB | Fixed bank (engine code + data tables) |
| $8000-$BFFF | 16KB | Switchable Bank 0 (song data) |
| $C000-$DFFF | 8KB | Switchable Bank 1 |
| $E000-$EFFF | 4KB | Switchable Bank 2 |
| $F000-$F7FF | 2KB | Switchable Bank 3 |
| $F800-$FFFF | 2KB | Work RAM |

### Key Z80 Vectors
| Address | Handler | Purpose |
|---------|---------|---------|
| $0000 | `DI; JP $0080` | Reset → init routine |
| $0008 | `RET` (trampoline) | RST 08 — YM2610 write helper |
| $0010 | `JP $1208` | RST 10 — YM2610 timing |
| $0038 | `DI; JP $1944` | IRQ (IM1) — Timer A tick |
| $0066 | NMI handler | 68K command reception |

### YM2610 Port Access
The driver uses `OUT (C), A` (opcode ED 79) for ALL YM2610 writes, NOT `OUT (n), A`.
This is critical — the port number comes from register C, not an immediate byte.

| Z80 Port | Direction | Function |
|----------|-----------|----------|
| $04 | Write | YM2610 Pair A address |
| $05 | Write | YM2610 Pair A data |
| $06 | Write | YM2610 Pair B address |
| $07 | Write | YM2610 Pair B data |
| $04 | Read | YM2610 Status 0 (Timer flags) |
| $06 | Read | YM2610 Status 1 (ADPCM end flags) |
| $00 | Read | 68K command byte |
| $0C | Write | Reply to 68K |
| $08 | Write | Enable NMI |
| $18 | Write | Disable NMI |
| $08-$0B | Read | Bank switching (upper addr bits = bank number) |

## Boot Sequence

1. Z80 starts at $0000 → jumps to $0080
2. Init routine: sets SP=$FFFC, IM 1, clears RAM
3. Keys off all FM channels (reg $28)
4. Silences SSG (reg $07 = $3F)
5. Dumps ADPCM-A channels
6. Sets up Timer A ($24/$25) and enables it ($27 = $35)
7. Enables NMI, enables interrupts (EI)
8. Enters main loop at $0107

## Main Loop ($0107-$0114)

```
$0107: EI
$0108: CALL $0115      ; check ring buffer for commands
$010B: LD ($FDE8), A   ; store current command
$010E: CALL $0140      ; dispatch command
$0111: EI
$0112: JP $0107        ; loop
```

The main loop spins continuously, checking the ring buffer for 68K commands.
NMI interrupts it to add commands; Timer A IRQ interrupts it for sequencer ticks.

## 68K Command Protocol

### NMI Handler ($0066)
When the 68K writes to REG_SOUND ($320000):
1. Z80 NMI fires
2. Handler reads command from port $00
3. Stores it in ring buffer at $FDF5 + write_index
4. Increments write pointer at $FE37 (masked to 6 bits = 64-entry buffer)
5. Echoes command on port $0C, clears on port $00
6. Returns (RETN)

### System Commands (processed in NMI, not ring buffer)
| Command | Handler | Function |
|---------|---------|----------|
| $01 | $0C58 | Slot switch / soft reset |
| $03 | $0C66 | Stop all sound, silence |
| $10 | $0C74 | Full reset |

### Music/SFX Commands (buffered, processed in main loop)
The main loop reads commands from the ring buffer and dispatches:
- Commands < $20: system sub-commands via table at $0C18
- Commands >= $20: type lookup via table at $2E6C

### Command Type Table ($2E6C)
224 entries (one per command $20-$FF):
- Type 0: unused/ignored
- Type 2: music track
- Type 5: ADPCM SFX sequence

**Active entries:**
- $20-$2E: Type 2 (15 music tracks)
- $40-$4D: Type 2 (14 music tracks)
- $70-$73, $7A, $7F: Type 5 (6 ADPCM entries)

### Low-Range Commands ($00-$1F) — Table at $0C18
| Cmd | Handler | Function |
|-----|---------|----------|
| $07 | $0EEE | **Unlock music** (clears $FDE1 lock flag) |
| $08 | $0EFA | Unlock music (alternate) |
| $14 | $0DCB | Two-byte SFX dispatch |
| $18 | $0D20 | ADPCM channel 0 SFX |
| $19 | $0D95 | ADPCM channel 6 SFX |
| $1A | $0D32 | ADPCM channel 1 SFX |
| $1B | $0D6B | ADPCM channel 4 SFX |
| $1C | $0D4D | ADPCM channel 2 SFX |
| $1D | $0D7A | ADPCM channel 5 SFX |
| $1E | $0D5C | ADPCM channel 3 SFX |

## Music Lock Flag ($FDE1)
- On boot: set to $FF (music LOCKED)
- Command $07 clears it to $00 (music UNLOCKED)
- Command $03 (stop) re-locks it to $FF
- The music handler at $102B checks $FDE1 and returns immediately if non-zero

**CRITICAL**: The 68K must send command $07 before any music commands will work.
After sending $03 (stop), must send $07 again to re-unlock.

## Song Structure

### Song Pointer Table ($30D0)
32 entries × 2 bytes (little-endian), indexed by (command - $20).
Each points to the song header in ROM.

### Bank Configuration
- Bank ID table at $3210: one byte per song, indexes into bank config table
- Bank config table at $267F: 4 bytes per entry (bank numbers for ports $08-$0B)
- Songs 0-6: bank config 0 (banks $1E/$0E/$06/$02)
- Songs 7-13: bank config 1 (banks $2E/$16/$0A/$04)
- Song 14: bank config 2 (banks $3E/$1E/$0E/$06)

### Validation Table ($2B38)
One byte per song slot. Must be $01 for valid songs. Working songs all have $01.

### Song Header (14 bytes)
| Offset | Size | Content |
|--------|------|---------|
| 0-6 | 7 | Channel enable flags (FM1-FM4, SSG-A/B/C) |
| 7-9 | 3 | Channel enable flags (ADPCM?) |
| 10 | 1 | Channel 11 enable |
| 11 | 1 | Tempo (Timer subdivision) |
| 12 | 1 | Speed/ticks-per-row |
| 13 | 1 | Config ($20 for standard) |

### Channel Pointer Table
Immediately after header. 2 bytes (LE) per active channel, pointing to that
channel's byte stream data in ROM.

## Channel State ($FAF4+)

Each channel has a 39-byte ($27) state block, stored starting at $FAF4.
IY register points to the current channel during processing.

| Offset | Purpose |
|--------|---------|
| IY+0 | Active state (0=off, 1=init, 5=running) |
| IY+1 | Duration counter ($FF = trigger immediately) |
| IY+8 | Channel type/config ($C1 for FM) |
| IY+10 | Sequence pointer low byte |
| IY+11 | Sequence pointer high byte |
| IY+12 | Sequence pointer copy (low) |
| IY+13 | Sequence pointer copy (high) |

## IRQ Handler — Timer A Tick ($1944)

1. Saves all registers (AF, BC, DE, HL, IX, IY)
2. Reads YM2610 status registers (ports $04 and $06)
3. Resets Timer A flag (reg $27 = $35)
4. Calls tempo/subdivision counter ($257C)
5. For each active channel: calls channel tick handler
6. Restores registers, RETI

### Channel Tick Handler ($02BB)
For each channel:
1. Loads channel state via lookup table at $2676
2. Decrements duration counter (IY+1)
3. When counter reaches 0: reads next event from sequence
4. Calls $0340 (event dispatcher) → $1F06 (command processor) → $03AA (note processor)
5. If note: calls $01E7 (key-on) to write frequency + key-on to YM2610
6. If rest: calls $027D (key-off)

### Note Event Format
From Z80 trace analysis, the byte stream after init commands follows this pattern:

```
[note_byte] [pitch_byte] [duration] [gap/rest]
```

Where:
- **note_byte**: Base pitch selector (e.g., $37=G, $3A=Bb, $3C=C, $39=A, $35=F)
- **pitch_byte**: Fine pitch / F-number ($78, $73, $6E, $69, $64)
- **duration**: How long the note plays ($0C=short, $18=medium, $24=long)
- **gap**: Rest between notes ($0A typical, $16 for longer gaps)

The note_byte and pitch_byte together determine the FM frequency registers:
- note_byte → looked up in table at $2B5E → determines octave/block
- pitch_byte → looked up in table at $29FE → determines F-number

Values $00 and bytes < $18 in the "note" position are rests or timing delays.

### Init Sequence (per channel)
```
$00 $E6     Rest + key-off flag
$C3 $15     CONFIG (set timing subdivision)
$EC $69     INSTRUMENT (set FM patch $69)
$EB $03     PAN (center)
$83 $00     ??? (special init)
$EC $69     INSTRUMENT again
$C7 $03     LOOP START (count=3)
... note data ...
$C8         LOOP END
... more note data ...
$FF         END OF CHANNEL
```

## ADPCM SFX System

### Channel Rotation
The driver manages 3 ADPCM-A channels (hardware channels 3-5) for SFX:
- Channel state at $FAB0 (ch3), $FAB5 (ch4), $FABA (ch5)
- YM2610 register offsets: $0B, $0C, $0D

Allocation logic ($0413):
1. Check channel 3 — if free (state byte = 0), use it
2. Check channel 4 — if free, use it
3. Check channel 5 — if free, use it
4. If all busy: compare priorities, steal lowest

### SFX Data Format
Each SFX entry is 6 bytes:
| Byte | Content |
|------|---------|
| 0 | Priority (higher = harder to steal) |
| 1-2 | ADPCM-A start page address |
| 3-4 | ADPCM-A end page address |
| 5 | Volume + pan flags |

### Two-Byte SFX Commands
The 68K sends pairs: channel selector ($1A, $1C, etc.) then sub-command.
Must be sent on separate frames — back-to-back writes lose the second command.

| Prefix | ADPCM Channel | SFX Data Table (IX base) |
|--------|--------------|--------------------------|
| $18 | Channel group 0 | $447C |
| $1A | Channel group 1 | $49AA |
| $1C | Channel group 2 | $4FAA |
| $1D | Channel group 5 | $5E38 |

## FM Patch Tables

### Frequency Table ($29FE)
12 entries × 2 bytes (LE), one per semitone:
```
C=618  C#=627  D=636  D#=645  E=655  F=664
F#=674  G=683  G#=694  A=704  A#=714  B=724
```

### Note Mapping Table ($2B5E)
128 entries, maps note bytes to packed (semitone<<4 | octave).
Valid range: $18-$77. Values outside this range = $FF (invalid/rest).

### FM Patch Lookup
- EC command stores patch ID at IX+$10
- Patch data accessed through table at $2FCF
- Complex bit-shifting lookup to select patch parameters
- Patches define all 4 FM operators (TL, AR, DR, SR, SL/RR, DT/MUL)

## Verified via Z80 Trace

All findings verified using `tools/z80_trace.py` — a Python Z80 emulator that
executes the KOF96 M-ROM binary instruction-by-instruction, replicating the
Geolith emulator's memory map, bank switching, and YM2610 port I/O.

Key verification results:
- Init completes in 704 instructions
- Song load (cmd $21): correctly reads pointer from $30D0, switches banks, sets up 7 channels
- Sequencer produces 660 key-on events over 5000 ticks
- FM frequency registers ($A0-$A7) written with correct F-number/block values
- FM operator patches loaded (136 register writes for TL/AR/DR/SR/SL_RR/DT_MUL)
- ADPCM-A SFX: confirmed channel rotation across channels 3-5 with priority comparison
